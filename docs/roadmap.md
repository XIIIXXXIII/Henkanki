# Henkanki Roadmap

## Phase 0: foundation
- Keep the repository large, explicit, and navigable.
- Maintain local-first, no-database defaults.
- Ship runnable Web and CLI seeds before adding heavy converters.

## Phase 1: Web/PWA MVP
- File info.
- Text converters: JSON, YAML, CSV, Markdown, HTML, TXT.
- Browser image converters through Canvas/WebCodecs where available.
- Offline application shell.

## Phase 2: CLI/Core
- Stable conversion graph.
- Batch conversion.
- Portable binaries for x86_64, ARM64, ARMv7, i686, RISC-V.

## Phase 3: Desktop/Mobile
- Desktop shell with drag and drop.
- Android/iOS share targets.
- Local settings and local job history only.

## Phase 4: Heavy formats
- PDF, archives, audio/video, office documents, ebooks.
- Prefer local native tools; server conversion remains optional.

## Phase 5: Ports and retro
- BSD, Haiku, illumos, ReactOS, legacy Windows, old Android/iOS research.
